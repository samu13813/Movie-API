Business Logic URL HTTP Method Request body data format Response body data format
Gets a list of all movies /movies GET None A JSON object holding data about all the movies by name
Gets a genre by name /movies/genre/[GENRENAME] GET None A JSON object showing a description about the genre
Gets the data of the director by name /movies/director/[DIRECTORNAME] GET None A JSON object holding data about the director, containing name and date of birth. Example:
{
  name: "Christopher Nolan",
  bio: "Director Biography"
  birth: 1970
  death: "--"
}
Gets a list of all users /users GET None A JSON object holding data about all users
Gets a user by name /users/[USERNAME] GET None A JSON object holding data about the user. Example:
{
  username: "Will12",
  password: "testing123"
  email: "Will12@gmail.com"
  birthday: "19-08-1999"
  favoriteMovies: "[ ]"
}
Allows new user to register /users POST A JSON object holding data about the user to add, structured like:
{
  username: "Sam",
  password: "usernamepassword",
  email: "usernameemail@gmail.com",
  birthday: "14-08-1989",
  favoriteMovies: [ ]
}

Allows the user to update their details /users/[NAME] PUT A JSON object holding the data to update, structured like:
{
  Username: "newUserName",
  password: "newPassword",
  email: "newEmail@gmail.com",
  birthday: "updatedBirthday",
}
User can insert 1 to 4 fields 		A text message idicating whether the username was succesfully updated.
Allows user to add a movie to their list of favorites /users/[NAME]/movies/[MOVIEID] POST None A text message indicating whether the movie was succesfully added
Allows user to remove a movie from their list of favorites /users/[NAME]/[MOVETITLE] DELETE None A text message indicating whether the movie was succesfully removed
Allows user to delete their username /users/[NAME] DELETE None A text indicating whether the username was succesfully deleted
